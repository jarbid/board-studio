# Kernel interpolation & fitting — extracted specs

Three behaviors ported from the legacy `cadcore` kernel, each pinned by a golden
fixture in `docs/specs/golden/interp.golden.json` (generated by the real legacy
kernel via `tools/golden-exporter`, `mainClass = goldenexport.InterpGoldenExporter`).
Units: cm, cm², cm³.

## 1. sLinear (station-linear / arc-length) cross-section interpolation

**Legacy source:** `cadcore/BezierBoardSLinearInterpolationSurfaceModel.java`
(`getPointAt`, `getCrosssectionAreaAt`), dispatched from `board/BezierBoard.java:1296`
(`getCrossSectionAreaAt`) and `:1190` (`getVolume`). Supporting spline math:
`BezierSpline.getPointByS` / `getPointByCurveLength` / `getSByNormalReverse` /
`getLengthByTangentReverse`.

**Behavior.** Unlike `controlPoint` (which scales one _blended_ section), sLinear
blends two _separately scaled_ bounding stations in arc-length space:

1. Clamp `x` to `[0.1, length-0.1]`. Let `c1 = previous`, `c2 = next` real station.
2. Scale each station's spline to the board's local target width/thickness at `x`,
   using `targetWidth = getWidthAtPos(x)`, `targetThickness = getThicknessAtPos(x)`,
   and each station's own width and **center thickness at x≈0**
   (`getValueAtReverse(ZERO) − getValueAt(ZERO)`, _not_ the first/last knot
   difference used by `controlPoint`'s `csCenterThickness`).
3. For arc-length parameter `s ∈ [0,1]`, map it into each scaled spline's
   `[sMin, sMax]` window, where the window ends come from `getSByNormalReverse` at
   the requested normal angles (`sMin = ONE` if `minAngle ≤ 0`,
   `sMax = ZERO` if `maxAngle ≥ 270`).
4. Sample `v1 = pointByS(c1Scaled, s1)`, `v2 = pointByS(c2Scaled, s2)` and blend
   linearly by `d = (x − pos1)/(pos2 − pos1)`, then add `getRockerAtPos(x)` to the
   height.
5. **Area:** trapezoid integral over `s∈[0,1]` of the blended point (lateral, height)
   for the deck (normal angles −90…90) minus the bottom (90…360), `×2`, clamped ≥ 0.
   The legacy sLinear `getCrosssectionAreaAt` uses a fixed `AREA_SPLITS = 10`
   internally and **ignores** the caller's `splits` argument; reproduced.

**Fixture (`sLinear`):** per board, length, total `getVolume()` (sLinear), and area
at x = {10,25,50,75,90}% of length, all computed with the legacy in sLinear mode.

**Tolerance:** the port reproduces the legacy to ~12 significant figures (same
root-finder / Simpson constants), so it is essentially exact. The test asserts a
comfortable `1e-4` relative band (measured max deviation: 0.000%). No intentional
numerical deviation.

## 2. Cross-section morph for differing control-point counts

**Legacy source:** `cadcore/BezierBoardCrossSection.java:179` (`interpolate`), using
`BezierKnot.compareTo` (`:329`, weights `COMPARE_POS_WEIGHT=4`,
`COMPARE_ANGLE_WEIGHT=15`, `COMPARE_TANGENT_LENGTH_WEIGHT=0.3`),
`BezierSpline.getSplitControlPoint` (`:902`) and `BezierCurve.getSplitControlPoint`
(de Casteljau, `:794`) / `getClosestT` (`:736`).

**Behavior.** After scaling the target to the source's center-thickness/width, if the
two splines differ in control-point count the **sparser** one is resampled until the
counts match:

1. Pick `most` = spline with more CPs, `other` = fewer. Compute
   `scaleX = otherMaxX/mostMaxX`, `scaleY = otherMaxY/mostMaxY`.
2. While `most` has more CPs: for each interior CP of `most`, find its best
   `compareTo` match among interior CPs of `other` (with `other` endpoints scaled by
   `scaleX/scaleY`, tangents untouched); track the CP whose best match is _worst_.
3. Insert a new CP into `other` near that worst CP's endpoint via de Casteljau split
   at the closest `t`, then fix up the two neighbours' inner tangents:
   `prev.next = prev.end + (prev.next − prev.end)·ct`,
   `next.prev = next.end + (next.end − next.prev)·(ct−1)`.
4. With equal counts, blend each CP linearly: `a + t·(b − a)` (so `t=0` → source,
   `t=1` → morphed target).
5. If no valid insertion point is found, the legacy bails and returns the source
   clone; reproduced.

**Fixture (`morph`):** a 4-knot source and 3-knot target (raw knots recorded), plus
the interpolated knots at `t = {0, 0.25, 0.5, 0.75, 1}`.

**Tolerance:** `1e-4` cm on every knot coordinate. The chained de Casteljau splits
and 32→…-split closest-t recursion match the legacy to that band. No intentional
deviation.

## 3. BezierFit (least-squares cubic fit)

**Legacy source:** `cadcore/BezierFit.java:701` (`bestFit`), `:794`
(`normalizedPathLengths`), `:768` (`M`).

**Behavior.** Given sample points, the t-parameters are the normalized cumulative
chord lengths `s∈[0,1]`. With design matrix `U` (rows `[s³, s², s, 1]`), data matrix
`D = [X|Y]`, and the cubic Bézier basis matrix `M`, the four control points are

```
P = M⁻¹ · (UᵀU)⁻¹ · Uᵀ · D
```

The legacy used UJMP and fell back to a symmetric-positive-definite inverse when a
determinant was zero. The port uses a Gauss-Jordan inverse with a Moore-Penrose
pseudo-inverse (normal-equations) fallback for the singular case, which equals
`invSPD` for the SPD matrices that arise here.

**Fixture (`bezierFit`):** two point sets (a smooth hump and a quarter-arc) with their
legacy-fitted control points.

**Tolerance:** `1e-5` cm per control-point coordinate (pure linear algebra; the only
difference vs. the legacy library is float rounding). No intentional deviation.

## Deferred

- sLinear `getDeckAt` / `getBottomAt` / `getNormalAt` (3D surface point/normal
  queries) are **not** ported — only the area/volume path the board model needs
  today. They reuse the same `sLinearPoint` machinery and can be added behind their
  own fixtures when the 3D renderer wants the sLinear surface.
- The legacy obsolete `LinearInterpolation` model is intentionally omitted (legacy
  marks it obsolete and keeps it only for reference).
